I'd like to start by fleshing out the overall vision for this repo, then we can start creating a plan to impelement that vision. The big picture goal is to create a pip-installable python package that serves as essentially a standardized "wrapper" around a variety of protein language models to make it much easier to use them, swap one model for another, and run benchmarking experiments on a bunch of different models. This would be oh so much easier if we had a single, unified API/CLI for interacting with all of the models, with the model-specific fiddly bits abstracted away behind the scnenes. I want to keep this repo very lightweight -- different pLMs have different version requirements for their dependencies (for example, one wants torch<=2.10 and another requires torch>=2.11. For this reason, I want all of the models themselves to be implemented as docker containers. The docker containers should be entirely standalone (even the model weights should be packaged into the container). That means that this repo doesn't need any of the ML dependencies required by the models, just the tooling necessary for functioning as the wrapper. Let's start by taking a first pass at drafting an overall VISION.md document (in `docs/VISION.md`) that will serve as the source of truth for the overall, high-level vision of the `plms` repo. Before we write anything down, let's discuss any questions you have.
