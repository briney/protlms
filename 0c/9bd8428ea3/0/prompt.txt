# ProtBERT — Technical Facts for Toolkit Integration

Researched from the ProtTrans paper, the agemagician/ProtTrans GitHub README, and the Rostlab HuggingFace model cards (plus the raw `config.json`, `vocab.txt`, and `special_tokens_map.json` on HuggingFace, which are the authoritative source for exact dimensions).

## 1. ARCHITECTURE

**Confirmed.** ProtBERT is a BERT-style bidirectional encoder trained with masked language modeling (MLM). Each protein sequence is treated as a complete document; **no next-sentence-prediction**; 15% of residues masked (80% `[MASK]`, 10% random, 10% unchanged) — standard BERT MLM.

- Model class: **`BertForMaskedLM`** — this is the value of the `architectures` field in `config.json`.
- Loadable via HuggingFace `transformers`. The model cards show `BertForMaskedLM`/`BertModel`/`BertTokenizer` explicitly. Because `config.json` declares `architectures: ["BertForMaskedLM"]` and it is a standard BERT, the `Auto*` classes (`AutoModelForMaskedLM`, `AutoTokenizer`, `AutoConfig`, `AutoModel`) all resolve correctly. (Note: the cards' code uses the explicit `Bert*` classes; `Auto*` is equivalent but not literally quoted in the card.)
- Sources: https://huggingface.co/Rostlab/prot_bert , https://huggingface.co/Rostlab/prot_bert/raw/main/config.json , https://github.com/agemagician/ProtTrans

## 2. CHECKPOINTS

Exact HuggingFace repo ids (both confirmed in the ProtTrans README "Models Availability" table):
- **`Rostlab/prot_bert`** — trained on **UniRef100** (~217M sequences)
- **`Rostlab/prot_bert_bfd`** — trained on **BFD** (Big Fantastic Database)

Architecture is identical for both (same `config.json` dimensions):

| Field | Value |
|---|---|
| hidden_size (embedding dim) | **1024** |
| num_hidden_layers | **30** |
| num_attention_heads | **16** |
| intermediate_size | **4096** |
| vocab_size (config) | **30** |
| max_position_embeddings | **40000** |
| total parameters | **~420M** |

Notes:
- **Params: ~420M** (from ProtTrans/Rostlab; consistent with the 1.68 GB fp32 weight file). The arXiv abstract page did not list the per-model count — the 420M figure comes from the model documentation/ProtTrans materials, not the abstract.
- **Vocab size nuance:** `config.json` says `vocab_size: 30`, but the model card / training docs describe an *effective* vocabulary of **21** (5 special tokens + 20 standard amino acids actively used). The `vocab.txt` actually contains 5 special tokens + 25 letters (the 20 standard AAs plus X, U, B, Z, O) = 30 entries. Use **30** for tensor shapes (the logits dimension), and be aware "21" in prose refers to the actively-trained tokens.
- **max_position_embeddings is 40000**, which is unusually large. The model was actually *trained* on sequences of length **512** (300k steps) and **2048** (100k steps). So while the position-embedding table technically allows up to 40000, practical/effective max length is ~512–2048; quality degrades well before 40000.

Other variants "worth including": the same paper ships **ProtT5** (`Rostlab/prot_t5_xl_uniref50`, `Rostlab/prot_t5_xl_bfd`), **ProtAlbert** (`Rostlab/prot_albert`), **ProtElectra**, and **ProtXLNet** — but those are different architectures. A community distilled BERT variant **`yarongef/DistilProtBert`** exists. For a "ProtBERT" entry specifically, the two canonical checkpoints are `Rostlab/prot_bert` and `Rostlab/prot_bert_bfd`.

- Sources: https://huggingface.co/Rostlab/prot_bert/raw/main/config.json , https://huggingface.co/Rostlab/prot_bert_bfd/raw/main/config.json , https://github.com/agemagician/ProtTrans

## 3. TOKENIZATION (critical)

**Confirmed on all points.**

- **Space-separated amino acids: YES.** Sequences are "uppercased and tokenized using a single space." The fill-mask example input is literally space-separated: `'D L I P T S S K L V V [MASK] D T S L Q V K K A F F A L V T'`. You must join residues with spaces (e.g. `" ".join(sequence)`) before tokenizing, otherwise the tokenizer treats the whole string as one/unknown token.
- **Rare AA mapping U, Z, O, B → X via regex: YES.** Exact line from the card:
  ```python
  sequence_Example = re.sub(r"[UZOB]", "X", sequence_Example)
  ```
- **Special tokens:** `[CLS]`, `[SEP]` added in standard BERT layout: `[CLS] seqA [SEP]` (or `[CLS] seqA [SEP] seqB [SEP]` for pairs). Full special-token set (from `special_tokens_map.json`): `pad_token=[PAD]`, `unk_token=[UNK]`, `cls_token=[CLS]`, `sep_token=[SEP]`, `mask_token=[MASK]`.
- **Tokenizer init:** `BertTokenizer.from_pretrained("Rostlab/prot_bert", do_lower_case=False)` — `do_lower_case=False` is required (amino acids are uppercase).

Exact feature-extraction snippet from the model card:
```python
from transformers import BertModel, BertTokenizer
import re
tokenizer = BertTokenizer.from_pretrained("Rostlab/prot_bert", do_lower_case=False)
model = BertModel.from_pretrained("Rostlab/prot_bert")
sequence_Example = "A E T C Z A O"
sequence_Example = re.sub(r"[UZOB]", "X", sequence_Example)
encoded_input = tokenizer(sequence_Example, return_tensors='pt')
output = model(**encoded_input)
```
- **Max sequence length / max_position_embeddings: 40000** in config; effectively trained at 512 / 2048 (see §2).
- Sources: https://huggingface.co/Rostlab/prot_bert , https://huggingface.co/Rostlab/prot_bert/raw/main/special_tokens_map.json , https://github.com/agemagician/ProtTrans

## 4. CAPABILITIES

- **(a) Per-residue embeddings via `output_hidden_states`: YES.** Standard BERT — call `model(**inputs, output_hidden_states=True)` or use `last_hidden_state` from `BertModel`, shape `(B, L, 1024)`. Strip the `[CLS]`/`[SEP]` positions to get per-residue vectors.
- **(b) Pooled embeddings (mean / CLS): YES.** Mean-pool `last_hidden_state` over real residue positions, or take the `[CLS]` token (index 0). (Note: BERT's `pooler_output` exists but is the tanh-projected CLS; for proteins mean-pooling is the common choice.)
- **(c) Per-position amino-acid logits for masked-marginal pseudo-log-likelihood: YES.** `BertForMaskedLM` outputs logits of shape `(B, L, 30)` over the vocab, which is exactly what masked-marginal / pseudo-perplexity scoring needs.
- **(d) Variant effect scores: YES, derivable** (not a built-in head). Compute via masked-marginal scoring from the MLM logits (log-ratio of mutant vs. wild-type residue probability at the masked position) — the standard ESM-style approach. ProtBERT provides the logits; the scoring logic is yours to implement.
- **Mask token for masked-marginal scoring: YES.** `mask_token = "[MASK]"`. A `fill-mask` pipeline is demonstrated in the card, confirming the MLM head and mask token work out of the box.
- Sources: https://huggingface.co/Rostlab/prot_bert , https://huggingface.co/Rostlab/prot_bert/raw/main/special_tokens_map.json

# Profluent-E1 — Technical Findings

Sources used: the bioRxiv paper abstract/metadata (DOI `10.1101/2025.11.12.688125`), the GitHub repo `https://github.com/Profluent-AI/E1` (README + `src/E1/` source), and the HuggingFace `Profluent-Bio/E1-{150m,300m,600m}` config/API. The biorxiv full-text HTML is 403-blocked to fetchers; the PDF (`.../v1.full.pdf`) and JATS XML are available if deeper paper detail is needed, but the abstract plus the open-source code already answer everything below.

## 1. ARCHITECTURE — Encoder (MLM), not autoregressive

E1 is a **bidirectional encoder / masked language model**, explicitly positioned as a "drop-in replacement for the ESM family." The model class is `E1ForMaskedLM` and prediction uses a `?` mask token.

Backbone is a custom transformer (NOT vanilla BERT/ESM), confirmed by the HF `config.json`:
- Objective/attention: **block-causal multi-sequence attention** for retrieval — quoting the paper abstract: "By integrating retrieved evolutionary context through block-causal multi-sequence attention, E1 captures both general and family-specific constraints without fine-tuning."
- Architectural features from config: `hidden_act: "silu"`, `gated_mlp: true` (SwiGLU-style FFN), RMSNorm (`rms_norm_eps`), RoPE with **two separate position systems** — `rope_theta_within_seq: 10000` and `rope_theta_global: 500000`, `clip_qkv: 8`, and **`global_attention_every_n_layers: 3`** (a local/global attention interleave, similar in spirit to ModernBERT). `tie_word_embeddings: false`.
- Source files confirming the custom stack: `src/E1/model/attention.py`, `ffn.py`, `flash_attention.py`, `flex_attention.py`, `varlen_flex_attention.py`.

The paper motivates this as fixing "parameter inefficiencies... baked-in phylogenetic biases, and functional performance issues" of single-sequence pLMs. Trained on **four trillion tokens from the Profluent Protein Atlas**.

## 2. SINGLE-SEQUENCE vs RETRIEVAL-AUGMENTED

Both modes use the **same model and the same input string**; the only difference is whether you prepend homologs, comma-separated:
- Single-sequence: `"AAAAA?C"` (one sequence, `?` = optional mask)
- Retrieval-augmented: `"MFCATEEKL,MCCASDF,MFCC?SEF"` (homologs prepended, comma-separated; the **last** comma-separated entry is the query). The `sequence_ids` from the batch preparer track which residues belong to which sequence; the query is selected via `sequence_ids == sequence_ids.max()`.

Minimal single-sequence call (low-level, verbatim from README, file `README.md`):
```python
import torch
from E1.batch_preparer import E1BatchPreparer
from E1.modeling import E1ForMaskedLM

model = E1ForMaskedLM.from_pretrained("Profluent-Bio/E1-300m").to("cuda:0")
model.eval()

sequences = ["AAAAA?C"]                      # single-sequence mode
batch_preparer = E1BatchPreparer()
batch = batch_preparer.get_batch_kwargs(sequences, device="cuda:0")

with torch.autocast("cuda", dtype=torch.bfloat16, enabled=True):
    outputs = model(
        input_ids=batch["input_ids"],
        within_seq_position_ids=batch["within_seq_position_ids"],
        global_position_ids=batch["global_position_ids"],
        sequence_ids=batch["sequence_ids"],
        past_key_values=None, use_cache=False,
        output_attentions=False, output_hidden_states=False,
    )
logits = outputs.logits          # (B, L, V)
embeddings = outputs.embeddings  # (B, L, E)
```

Higher-level helpers (files `src/E1/predictor.py`, `src/E1/scorer.py`):
- `E1Predictor(model).predict(sequences, ..., context_seqs=None)` yields dicts with `"token_embeddings"` and `"mean_token_embeddings"` (mean-pool). `context_seqs` is the retrieval hook; omit it for single-sequence.
- `E1Scorer(model, method, ...).score(parent_sequence, sequences, ..., context_seqs=None, context_reduction="mean")` for variant scoring.

CLI tools also exist: `src/E1/tools/predict.py` and `src/E1/tools/score.py`. Cookbook notebooks: `cookbook/basic.ipynb`, `embedding.ipynb`, `zero_shot_fitness_prediction.ipynb`, `site_saturation_mutagenesis.ipynb`.
